# 空沙盘与即时建筑编辑器设计

## 背景

当前应用存在四个相互关联的问题：

1. 首屏静态显示“场景已准备好 / 打开示例项目”，项目载入后该提示可能仍覆盖场景。
2. 左侧项目对象是静态占位内容，不能选中真实建筑或进入编辑。
3. 时间轴使用独立控制器，未把日期和时间变化同步给 Three.js 太阳光，因此进度条变化时阴影不变化。
4. “新建项目”采用五步全屏向导，用户需要先完成大量设置才能看到结果；向导又创建独立项目，容易覆盖已有草稿。

本次改造把产品主线调整为“打开即是沙盘，在沙盘上持续编辑”。示例项目不再作为首屏引导，新建向导退出主流程，建筑、时间和日期都在同一页面即时预览。

## 目标

- 新用户打开应用时看到空沙盘，不自动加载示例项目。
- 已有本地草稿的用户刷新后恢复离开时的建筑和编辑状态。
- 沙盘常驻 10 米网格、方向指南针、坐标原点和比例说明。
- 添加建筑时立即在主沙盘显示半透明蓝图，不跳转到独立页面。
- 建筑参数和 X/Y 坐标变化时，蓝图实时响应。
- 左侧对象树来自真实项目数据，能够选中并重新编辑建筑。
- 日期和时间分别支持拖动与自动播放，太阳位置和阴影实时变化。
- 只有用户明确执行“清空沙盘”并二次确认后，才删除场景对象。

## 非目标

- 本次不新增地图服务、真实地块边界、CAD/BIM 导入或多人协作。
- 网格只作为尺寸参考，不提供强制吸附。
- 本次不重做观察区、窗户和分析算法，只保证它们继续挂在建筑对象下并兼容新的编辑流程。
- 本次不更改项目默认城市和太阳位置算法。

## 采用方案

采用“轻量编辑模式 + 单一项目状态”方案。

不继续扩展现有五步向导，也不全面重写编辑器。项目状态成为建筑编辑、对象树、Three.js 场景、结果面板、日期时间控制和本地保存的共同数据源。这样能够直接解决现有状态分裂问题，同时保留已有领域模型、场景构建和分析能力。

## 用户体验

### 首次进入与恢复

- 首次进入时创建默认空项目，沙盘中没有建筑。
- 主视口不再显示“打开示例项目”的覆盖卡片。
- 空场景仅显示不遮挡操作的短提示，例如“点击左侧添加建筑开始布置”。
- 如果本地已有有效草稿，则直接恢复草稿，不弹出向导。
- 示例 JSON 仍可保留为测试数据或导入资源，但不作为首屏主操作。

### 沙盘坐标与方向

- 沙盘中心定义为 `(0, 0)`。
- 面向沙盘平面的用户坐标采用 X/Y：
  - `+X` 指向东。
  - `+Y` 指向北。
- UI 坐标单位为米，允许小数。
- Three.js 内部继续使用 X/Z 地平面，UI 的 Y 坐标在场景适配层映射到 Three.js Z 轴。
- 网格主间距为 10 米，仅供视觉参考。
- 指南针常驻视口角落，并明确标注北向。
- 原点和正方向在编辑蓝图时可增强显示，普通浏览时保持弱化，避免视觉噪声。

### 添加建筑

1. 用户点击左侧“添加建筑”。
2. 应用立即创建带默认参数的建筑，并在主沙盘显示半透明蓝色蓝图。
3. 右侧打开轻量建筑参数面板，默认选中该建筑。
4. 用户修改建筑名称、模板、长度、进深、楼层数、层高、X、Y 和旋转角度。
5. 每次合法输入都立即更新项目状态和蓝图。
6. 用户点击“完成建筑”退出蓝图状态，建筑恢复实体材质。

“完成建筑”不是提交项目的唯一时机。新增建筑及其每次合法修改都会自动保存，因此刷新不会丢失正在编辑的蓝图。若用户选择“取消本次添加”，只删除本次新建建筑；编辑已有建筑时不显示该取消操作。

### 重新选择与编辑

- 左侧对象树由 `project.buildings` 及其楼层、观察区和窗户动态生成。
- 点击左侧建筑条目或沙盘中的建筑，都会选中同一对象。
- 选中建筑后右侧显示其参数；进入编辑时使用蓝图材质。
- 修改参数会递增建筑修订号，使场景同步器重建对应几何体。
- 完成编辑后保留选中高亮，但退出蓝图材质。
- 删除单栋建筑必须提供明确操作，并对包含观察区或窗户的建筑进行确认。

### 清空沙盘

- 原“新建项目”主操作改为“添加建筑”。
- 顶部提供低强调度的“清空沙盘”操作。
- 清空前必须二次确认，并明确将删除建筑、观察区和窗户。
- 清空只清除场景对象，保留城市、日期、时间、镜头和质量偏好。
- 确认后立即保存空项目，刷新后仍为空沙盘。

## 日期与时间

### 日期控制

- 新增全年日期滑条，范围为当年 1 月 1 日至 12 月 31 日，步长为一天。
- 提供日期输入框用于精确选择。
- 标注春分、夏至、秋分、冬至等参考节点，但节点不改变计算逻辑。
- 日期播放默认逐日推进，到年末后循环到年初。
- 播放日期时保持当前时间不变。
- 用户手动拖动日期滑条时暂停日期播放。

### 时间控制

- 保留日内时间滑条与时间输入框。
- 时间范围使用当天实际日出与日落；日期或地点变化时重新计算范围。
- 时间播放只推进时间，保持日期不变。
- 用户手动拖动时间滑条时暂停时间播放。

### 太阳和阴影同步

日期、时间或地点变化后：

1. 使用同一项目状态计算太阳高度角、方位角和方向。
2. 更新 Three.js 方向光的位置、目标和强度。
3. 标记相关阴影贴图需要更新。
4. 更新太阳方向指示、当前结果和全天分析请求。

太阳位于地平线以下时，关闭直射方向光和阴影，并将当前状态显示为夜间，避免出现虚假的夜间阴影。

## 界面结构

桌面端维持三栏布局：

- 左栏：场景对象树和“添加建筑”主按钮。
- 中栏：Three.js 主沙盘、网格、指南针、坐标提示和蓝图。
- 右栏：当前选中对象的轻量参数面板；无选中对象时显示结果或简短操作提示。
- 底部：日期滑条和时间滑条各占一行，分别拥有播放键和精确输入。

移动端继续使用底部导航和抽屉：

- 沙盘保持主画面。
- 添加建筑和对象列表进入“场景”抽屉。
- 参数面板使用可折叠底部抽屉，不覆盖整屏。
- 日期与时间控制可在底部压缩显示，并保留触屏可用的滑块尺寸。

## 状态与数据流

项目状态是唯一持久化数据源，至少包含：

- 地点与时区。
- 建筑列表及建筑修订号。
- 模拟日期、时间和当前观察区。
- 当前选中建筑和编辑状态。
- 镜头及界面偏好。

交互流程统一为：

`用户输入 -> 项目状态 -> 场景 / 对象树 / 参数面板 / 结果 -> 本地保存`

应用挂载时只创建一份项目状态。建筑编辑器、时间轴和结果控制器不再各自维护互不相通的日期、时间或建筑副本。

本地保存采用短防抖，连续拖动时避免频繁写入。页面卸载前若仍有待保存变化，则同步执行最后一次保存。项目导入会替换当前项目并清空撤销历史，随后立即刷新所有消费者。

## 组件边界

### 项目编辑状态

负责选中对象、蓝图模式、添加取消和完成编辑。它不直接操作 Three.js 对象，只更新项目状态。

### 建筑参数面板

负责表单呈现、输入校验和合法值提交。坐标显示为 X/Y，场景轴映射不进入表单层。

### 动态对象树

只根据项目状态生成层级，并发出选择、添加和删除意图。它不保存建筑副本。

### 场景同步器

根据建筑 ID 和修订号创建、更新或删除网格对象。根据当前编辑 ID 选择实体材质或蓝图材质。

### 沙盘辅助层

负责网格、比例说明、指南针、坐标原点和方向标记。辅助层不参与日照遮挡计算和对象拾取。

### 太阳光控制器

订阅地点、日期和时间，计算并更新 Three.js 太阳光与太阳方向覆盖层。它与结果计算使用同一太阳位置数据。

### 日期与时间控制器

负责两个独立播放状态。任一播放只修改自己的维度；手动输入会暂停对应维度。

### 草稿持久化

订阅项目变化并防抖保存。保存或恢复失败时发出可读错误，不让异常中断编辑器。

## 输入校验与异常处理

- 建筑长度、进深、层高和楼层数必须为正数。
- 楼层数取合法整数；旋转角度规范到 `0–359.9°`。
- X/Y 坐标必须为有限数值，允许负数和小数。
- 输入处于空白或非法的中间态时，字段显示错误，但不覆盖最后一个合法项目值。
- 不支持 WebGL 时继续显示现有降级提示，页面其余项目操作仍可使用。
- 本地存储不可用、容量不足或草稿损坏时显示提示；损坏草稿被移除后回到默认空项目。
- 导入无效项目时保留当前项目，不进行部分替换。
- 全天分析的旧请求结果不得覆盖较新的日期、时间或建筑状态。

## 测试设计

### 单元测试

- 默认项目为空且坐标约定稳定。
- UI X/Y 与 Three.js X/Z 的映射。
- 建筑新增、编辑、取消新增、删除和清空命令。
- 建筑修订号在影响几何体的参数变化后递增。
- 对象树从项目状态生成并正确选择对象。
- 日期的全年范围、闰年范围和循环播放。
- 日期播放不修改时间；时间播放不修改日期。
- 日期、时间和地点变化会产生新的太阳方向。
- 夜间关闭方向光与阴影。
- 本地草稿恢复、损坏草稿处理和清空后的保存。

### 端到端测试

- 新浏览器首次进入显示空沙盘、网格和指南针，不显示示例覆盖卡片。
- 添加建筑后立即出现蓝图。
- 修改 X/Y、尺寸、楼层和角度时蓝图实时变化。
- 完成建筑后显示实体材质。
- 点击对象树条目和沙盘建筑都能重新打开对应参数。
- 刷新页面后建筑、坐标、日期和时间保持不变。
- 日期滑条及播放会改变太阳方向和阴影；时间轴亦然。
- 播放一个维度时另一个维度保持不变。
- 清空操作取消时不改变项目，确认时只删除场景对象。
- 桌面 Chrome 与 Pixel 7 项目均可完成主要流程，支持键盘和触屏。

### 完成前验证

依次运行：

```text
npm test
npm run test:e2e
npm run build
```

任何测试失败或构建警告都必须在交付前解释或修复。

## 验收标准

- 首屏没有示例项目覆盖提示，空沙盘可直接操作。
- 网格、比例和指南针始终可见且不参与遮挡计算。
- 建筑参数及 X/Y 坐标变化能在同一主沙盘实时预览。
- 左侧对象树点击有效，并与沙盘选择保持同步。
- 日期和时间各自可拖动、播放，另一维度保持固定。
- 每次日期或时间变化都会驱动太阳光和阴影更新。
- 刷新不会丢失已添加建筑或正在编辑的蓝图。
- 只有经用户二次确认的清空操作会删除全部场景对象。
