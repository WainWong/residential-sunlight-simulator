# 住宅采光模拟器设计规格

日期：2026-07-02  
状态：已完成产品讨论，等待用户审阅

## 1. 产品目标

本项目是面向普通用户的住宅采光模拟工具。用户无需掌握 CAD 或完整户型建模，只需用建筑模板搭建场景、选择楼层、涂选关注区域并设置采光界面，即可判断某日某时是否存在太阳直射、阳光照到哪里，以及当天的直射时段和总时长。

产品提供购房参考级结果：太阳位置和建筑遮挡计算应可靠、过程可解释，但产品必须明确说明，结果不能替代专业日照合规报告。

项目为纯前端静态应用，不需要账号、数据库或自建后端，可部署到 Vercel，并适合在 GitHub 开源协作。

## 2. 第一版范围

### 2.1 正式包含

- 多建筑场景；所有建筑使用统一能力模型。
- 一字型、L 型、回字形三种参数化模板。
- 建筑尺寸、位置、旋转、楼层数、标准层高和可选首层层高。
- 任意建筑、任意楼层上的多个观察区域。
- 通过 1 米网格点击或拖动涂选观察区域。
- 普通窗、落地窗、阳台三种采光界面。
- 常用城市搜索选择，以及经纬度和时区的手动设置。
- 日期与时间控制、太阳高度角和方位角。
- 当前时刻直射判断、被照亮面积比例和可视化。
- 当天直射时间段、总直射时长和时间轴。
- 一天播放、速度控制、日出和日落跳转。
- JSON 项目导入与导出。
- 当前画面截图导出。
- 桌面和手机端完整建模、模拟、保存与导入能力。

### 2.2 明确不包含

- CAD、BIM、GIS 或真实地图建筑导入。
- 自由户型绘制和装修级室内建模。
- T 型、U 型、点式塔楼和连廊楼模板。
- 小窗、连廊窗、天井窗和封闭阳台专用模型。
- 真实玻璃折射、复杂窗框和植被动态遮挡。
- 全年日照统计、多区域并行对比和专业规范判定。
- 用户账号、云端保存和协作编辑。

后续增加楼型或采光界面时，应通过注册新模板实现，不修改既有项目数据结构。

## 3. 核心产品模型

所有建筑均为相同的 `Building` 类型，不区分“目标建筑”和“遮挡建筑”。每栋建筑既可承载楼层、观察区域和采光界面，也会参与其他区域的遮挡计算。

一个项目可以包含多栋建筑和多个观察区域，但模拟界面一次只激活一个观察区域。这样可以控制实时计算成本，同时为后续多区域对比保留数据基础。

核心实体如下：

- `Project`：版本、名称、地点、建筑集合、模拟配置和视图状态。
- `Location`：城市标识、纬度、经度和 IANA 时区。
- `Building`：模板类型、参数、位置、旋转、楼层和外观设置。
- `ObservationArea`：所属建筑与楼层、网格单元、名称和采样高度。
- `Opening`：所属建筑与楼层、外墙边、类型、位置和尺寸。
- `SimulationConfig`：当前日期时间、激活区域、采样配置和播放速度。
- `ViewState`：相机、面板和当前编辑模式。

JSON 根对象必须包含 `schemaVersion`。导入时先校验，再按版本执行迁移；无法迁移的项目不写入当前状态，并展示可读错误。

## 4. 用户流程

### 4.1 首次创建

采用五步引导式流程：

1. 设置地点。
2. 添加并摆放建筑。
3. 选择楼层并涂选观察区域。
4. 在外墙上添加采光界面。
5. 进入模拟并查看结果。

每一步只展示完成当前任务所需的参数。用户可以返回上一步，已输入数据不会丢失。

### 4.2 日常编辑

完成首次引导后进入自由工作台。用户可以直接切换建筑、楼层、观察区域、日期和时间，不必重新通过向导。再次打开导入的项目也直接进入工作台。

## 5. 响应式界面

### 5.1 桌面布局

- 顶部栏：项目名称、撤销/重做、导入、保存和截图。
- 左侧栏：建筑、楼层、观察区域和采光界面的项目树。
- 中央：占据主要空间的 3D 场景。
- 右侧栏：当前对象参数与分析结果，可切换页签。
- 底部：日期、时间轴、播放和日出日落操作。

左右侧栏均可折叠。3D 场景在窗口尺寸改变后自适应重绘。

### 5.2 手机布局

- 3D 场景保持为主画布。
- 项目树、对象参数和结果进入可升降的底部抽屉。
- 底部主导航切换建筑、区域、模拟和结果。
- 时间轴保持靠近画布，可单手拖动。
- 建筑和区域编辑使用分步页面，避免小屏同时显示过多控件。

桌面和手机共享同一套状态与功能，不建立功能缩水的移动版。

### 5.3 视觉与可访问性

视觉基调为深石墨、暖阳黄和米白，营造克制的建筑工具感。不得使用大面积赛博蓝数据大屏风格。

- 直射、遮挡和错误状态同时使用颜色、文字与图标表达。
- 触控目标最小尺寸为 44×44 CSS 像素。
- 表单、时间轴和主要命令支持键盘操作。
- 尊重 `prefers-reduced-motion`，关闭非必要动画。
- 面板与画布需要保持可读对比度。

## 6. 技术架构

项目采用 Vite、原生 JavaScript ES 模块和 Three.js。Vite 只负责开发与静态构建，产物可直接部署到 Vercel。

代码职责边界：

- `src/domain/`：不依赖 DOM 或 Three.js 的建筑、太阳、几何和模拟计算。
- `src/scene/`：Three.js 场景、相机、控制器、建筑网格、阴影与选择表现。
- `src/features/`：建筑编辑、楼层选择、观察区涂选、开口编辑、时间轴、结果、导入导出。
- `src/store/`：唯一项目状态、命令、派生状态和订阅机制。
- `src/workers/`：全天直射扫描。
- `src/ui/`：通用组件、响应式外壳、对话框和通知。

每个模块须具备清晰输入输出。`domain` 只能接收普通数据并返回普通数据，不直接操作场景或界面；`scene` 不拥有业务真相，只渲染状态。

## 7. 数据流

数据流保持单向：

1. 用户操作产生带校验的命令。
2. 命令更新唯一项目状态。
3. 状态变化标记受影响的建筑、区域或模拟配置。
4. 场景层只重建受影响的几何体。
5. 当前时刻结果同步更新。
6. 全天结果由 Web Worker 异步计算。

相机旋转、平移和缩放只修改 `ViewState`，不触发采光重算。全天计算任务附带递增的请求编号；收到结果时只有最新编号可以写回状态，旧任务结果被丢弃。

## 8. 几何与采光计算

### 8.1 太阳位置

根据纬度、经度、IANA 时区、日期和当地时间计算太阳高度角、方位角与世界坐标方向向量。夏令时由时区规则处理，不允许用固定 UTC 偏移替代时区。

太阳位于地平线以下时，直接返回无直射，不执行射线检测。

### 8.2 观察区域采样

观察区域由 1 米网格单元组成。每个单元使用中心点和均匀分布的附加点采样；第一版默认每平方米四个采样点。每个采样点具有统一的用户可调离地高度，默认 0 米。

当前直射条件为：至少一个有效采样点收到直射。被照亮面积比例为收到直射的采样点数量除以有效采样点总数。

### 8.3 开口与遮挡

每个观察区域关联一个或多个采光界面。对每个采样点向太阳方向发射射线：

1. 射线必须正向穿过一个已关联开口的有效多边形。
2. 射线穿过开口后，检测阳台顶板、侧板、本栋外部结构及场景内其他建筑。
3. 任一关联开口提供无阻挡路径，则该采样点收到直射。

此算法使用开口平面作为光线门户，不依赖对整栋建筑执行实时布尔挖洞。视觉层可以绘制窗洞效果，但计算层以门户几何为准。

阳台模型包含深度、宽度、顶板开关和栏杆高度。第一版不计算玻璃折射或透光率衰减。

### 8.4 全天分析

从当地日出扫描到日落，先按 5 分钟间隔计算。当相邻样本的直射状态不同，再在该时间段内细化到 1 分钟。输出直射区间和总时长，精度声明为约 1 分钟。

拖动时间时优先计算当前帧；全天分析在 Worker 中执行，并可被更新后的任务取代。

## 9. 参数校验与错误处理

- 建筑尺寸、层数、层高和开口尺寸必须为有限正数，并受合理上限约束。
- L 型和回字形的内轮廓必须保留有效墙体厚度。
- 观察网格只能位于所属楼层轮廓内。
- 开口必须落在选定外墙边内，不得跨越墙角或楼板。
- 导入文件必须通过结构、版本和数值范围校验后才能替换当前项目。
- 文件损坏或版本不支持时保留当前项目，并展示具体原因。
- WebGL 不可用时显示兼容性说明和建议，不呈现残缺工作台。
- Worker 出错时保留当前时刻模拟，全天结果区域显示可重试状态。

错误信息应指出对象与字段，例如“建筑 A 的中庭宽度必须小于外轮廓宽度”，不能只显示“参数无效”。

## 10. 保存、隐私与恢复

应用默认不上传项目数据。保存操作下载带版本号的 JSON；导入只在浏览器本地解析。

为防止误关闭，可将最近一次编辑状态写入 `localStorage` 作为本机草稿。首次进入时说明该行为，并提供清除本地草稿入口。正式可移植数据仍以 JSON 文件为准。

截图导出只包含 3D 画布和必要的日期、时间、地点水印，不包含展开的编辑面板。

## 11. 性能目标

- 中端手机在包含 10 栋常见参数化建筑的场景中保持可交互的相机操作。
- 参数拖动期间采用预览质量，操作结束后恢复完整阴影与分析。
- 建筑网格按模板复用材质和几何缓存；楼层线使用合并或实例化绘制。
- 只为当前激活区域计算实时采样；非激活区域只显示轮廓。
- 全天扫描不得阻塞主线程。

性能目标以实际设备测试为准；若低端设备压力过高，可降低阴影贴图和采样点显示质量，但不得静默降低计算时间精度。

## 12. 测试策略

### 12.1 单元测试

- 太阳高度角、方位角、日出和日落。
- 三种建筑模板的轮廓和高度。
- 坐标变换、楼层高度和旋转。
- 射线与开口相交、朝向判定和遮挡。
- 观察网格采样与面积比例。
- 全天区间合并与边界细化。
- JSON 校验、导入和版本迁移。

### 12.2 场景基准测试

至少固定以下案例及期望结果：

- 无遮挡南向窗。
- 太阳位于地平线以下。
- 低楼层被前方建筑完全遮挡。
- 遮挡边缘导致部分观察区域被照亮。
- 回字形中庭内侧开口。
- 阳台顶板遮挡高角度阳光。
- 多个开口中仅一个提供有效光路。

### 12.3 浏览器流程测试

桌面和手机视口分别覆盖：

- 从空项目完成五步引导。
- 添加第二栋建筑并验证遮挡结果变化。
- 编辑观察区域和开口。
- 拖动与播放时间轴。
- 导出 JSON，再导入并恢复场景。
- 导出截图。
- 非法导入文件和无 WebGL 环境。

## 13. 开源与部署

- 使用清晰的 README 说明本地开发、构建、测试、Vercel 部署和精度边界。
- 提供示例项目 JSON，便于体验和回归测试。
- 许可证在发布前由维护者选择；代码中不得预设未经确认的许可证。
- Vercel 使用静态构建输出 `dist/`，不依赖服务端函数。
- 依赖应尽量少，并锁定版本；太阳与几何核心算法须有测试，避免升级后静默改变结果。

## 14. 验收标准

第一版达到 Beta 级“八成完成度”时，应满足：

- 新用户可在不阅读专业文档的情况下完成一次建模与模拟。
- 三种楼型、多建筑、观察区涂选和三种采光界面完整可用。
- 桌面和手机均能完成全部主流程。
- 当前直射与全天时段来自几何计算，不依赖 Three.js 视觉阴影读取。
- 固定基准案例结果稳定，关键计算具备自动测试。
- JSON 导入导出可无损恢复项目。
- 页面可构建为静态产物并部署至 Vercel。
- 产品内明确展示“购房参考，不能替代专业日照合规报告”的说明。
