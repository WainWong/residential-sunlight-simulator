# 统一几何:建筑壳 CSG 真挖洞(设计)

日期:2026-07-10
状态:已与用户确认
前置:2026-07-10-interior-daylight-design.md(B+C 室内日光,已交付)

## 目标

建筑的渲染几何成为单一真理源:室内视角、室外视角、阴影投射全部使用同一套
"真正挖好洞、掏好房间"的网格。删除上一轮为绕开整体拉伸限制而引入的三层胶水:

- `createInteriorFloor`(独立房间网格:地板/天花板/薄盒隔墙/洞口留空)
- `createHostShadowGhost`(隐形投影替身:下段/上段/同层带孔板)
- `openingOverlay`(室外视角的洞口贴片 + 6cm 法线偏移防闪烁)

为后续漫反射 GI 铺路:GI 需要封闭、一致的场景几何,贴片和替身都是它的毒药。

## 方案选择

三个候选:A 解析构造(直接拼出挖完洞的矩形件)、B 三维布尔运算库
(three-bvh-csg)、C 降维布尔(2D 多边形/1D 区间减法后拉伸)。用户在了解
三维 CSG 的碎面与共面风险后仍选择 **B**,理由是表达力与未来异形洞的扩展性。
共面风险通过下述"分段组合 + 刀外扩"在设计层面全部消除。

## 几何组成:一栋楼 = 若干段上下叠放的实体

单一管线 `buildBuildingSegments(building)`,所有建筑(有无观察区)都走它:

1. 收集被观察区占用的楼层,得到高度切点列表;
2. 楼高按切点切成若干段,每段生成 footprint 拉伸体;
3. 有观察区的段执行 CSG 减法(three-bvh-csg `Evaluator`,SUBTRACTION);
   无刀的段减法列表为空,布尔自然跳过。

无观察区建筑是退化情形:零切点 → 一段 → 零次布尔,与现状渲染结果一致。
没有 if/else 双路径。

观察层段的构成:

- **下段** 0 → 观察层楼板顶(楼板厚 0.15m 归下段):整段拉伸,不动刀。
- **观察层段** 楼板顶 → 上层楼板底:刀 = 观察区多边形棱柱
  (rectUnionToPolygons 的并集轮廓,含内岛)。
- **上段** 上层楼板底 → 楼顶:整段拉伸,不动刀。

刀削剩的实体自然成为:隔墙(区边界的内部边)、同层其余实心部分、洞
(区边界与 footprint 墙重合的边,见下)。房间的地板 = 下段顶面,天花板 =
上段底面,全部是真几何,可投影可接影。回形观察区的内岛由 CSG 直接表达,
上一轮 Shape.holes 表达不了内岛导致的漏光随之消失。

## 共面风险的消除(设计不留任何一处精确共面的布尔)

三维布尔在两个面精确共面处做"里/外"分类是浮点 0/0,会产生闪烁膜或裂缝。
逐处消除:

| 潜在共面 | 消除方式 |
|---|---|
| 刀侧面 vs 外墙面(洞) | 刀在与 footprint 墙重合的边上向外扩 `OPENING_CUT_EXTENSION = 0.5m`,明确穿出墙外;多切的是空气,视觉无痕 |
| 刀底面 vs 楼板顶 / 刀顶面 vs 天花板底 | 刀的 y 区间上下各超出观察层段端面(段外无材料可切) |
| 段与段接缝 | 不同网格贴合摆放,不参与布尔,无分类问题 |

判定"区边界边是否在 footprint 墙上"复用现有 `edgeOnFootprint` 逻辑
(轴对齐共线 + 包含)。内部边不外扩:刀的终止面削出的立面就是隔墙表面,
它与既有几何无重合对象,不构成布尔歧义。

`OPENING_CUT_EXTENSION` 为命名常量;实现完成后按用户要求演示 0 / 0.05 /
0.5 三档对比(预期 >0 的档位视觉完全一致,0 档存在间歇性共面风险),
确认后保持 0.5。

## 进入室内的行为

- 相机飞入(`flyToArea`)不变。
- 不再隐藏建筑网格、不再添加房间/替身。
- 遮挡处理按段:相机低于天花板时,上段按现有"相机高度规则"淡出;观察层段
  墙体挡视线时沿用 raycast + occlusionFade,但淡出对象是整段材质(单一 CSG
  网格无法只淡一面墙)。第一版接受"整段淡出";若体验不佳,后续把观察层段
  按墙拆子网格。
- 金色洞口描边保留,贴在真洞边缘(从刀与墙的重合边生成 LineLoop)。
- 室外视角直接看见真洞,`openingOverlay` 删除。

## 不变的部分

- 数值分析完全不动:worker、`firstBlockingDistance`、`buildAreaWallQuads`、
  结果面板。数字归 CPU 解析射线,画面归 GPU,渲染几何与分析几何解耦。
- 太阳光/阴影相机聚焦(`frameShadowsOnInterior`)/hemisphere 调光逻辑保留,
  作用对象从"房间网格"换成分段实体。
- 拾取:段网格沿用 `userData.entityId = building.id`,picking 不改。
- 楼层参考线(floorLines)、选中高亮、预览蓝图材质行为不变(材质应用到
  每段上)。

## 依赖与重建时机

- 新增依赖 `three-bvh-csg`,安装时验证与当前 three 版本兼容。
- 几何仍按 `${revision}:${isPreview}` 签名 diff 重建;布尔只在建筑参数/
  观察区变化时执行,不进渲染循环。单段(一层楼)输入规模小,预计毫秒级;
  若可感知再防抖,先不过度设计。

## 测试

- 单测(node 环境可直接跑 three + three-bvh-csg 几何):
  - 切点/分段 y 区间正确;无观察区建筑输出单段;
  - 刀在洞边外扩、内部边不外扩;
  - 观察层段布尔输出水密(开放边界边数为 0)且三角形数量合理;
  - 回形观察区(含内岛)输出不缺面。
- e2e:沿用 interior-daylight.spec;补"室外视角下洞可见"断言。
- 手动验收:上一轮问题场景逐一复查(洞口闪烁、天花板时有时无、地面阴影
  裂缝),加 0 / 0.05 / 0.5 外扩对比演示。
